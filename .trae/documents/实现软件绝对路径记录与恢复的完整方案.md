## 方案微调：保留原字段，新增专用路径字段

你的建议非常稳妥。我们**不动现有的 `app_name`**（它用于显示、统计、图谱，简短好用），而是**新增一个 `app_path` 字段**专门用来存绝对路径。

### 修正后的实施方案

1.  **数据库变更 (Database)**
    *   创建新迁移文件 `0007_add_app_path.sql`。
    *   `ALTER TABLE activity_logs ADD COLUMN app_path TEXT;`
    *   这样旧数据该字段为 NULL，新数据开始记录路径，完全兼容。

2.  **采集层变更 (Recorder)**
    *   **WindowInfo**：修改结构体，新增 `process_path: String`。
    *   **Windows 实现**：在 `window_info.rs` 中，`QueryFullProcessImageNameW` 获取到的完整路径，赋值给 `process_path`；然后像原来一样提取文件名赋值给 `process_name`。
    *   **入库**：修改 `db::insert_activity`，多接收一个 `app_path` 参数并写入数据库。

3.  **AI 分析层变更 (Agent)**
    *   在构建 Context 给 AI 时，格式化为：`应用: chrome.exe (路径: C:\...\chrome.exe) | 标题: ...`。
    *   这样 AI 就能看到绝对路径了。
    *   Prompt 增加：如果识别到关键应用，提取其路径到 `related_apps`。

4.  **执行层变更 (Executor)**
    *   新增 `AutomationStep::OpenApp { path: String }`。
    *   执行逻辑：直接运行该路径。

### 验证方式
1.  修改代码后，重启应用（触发数据库迁移）。
2.  录制几个操作。
3.  查看数据库（或日志），确认 `app_path` 字段已有值。
4.  生成回顾，看 AI 是否能提取出这个路径并生成“启动应用”的步骤。
